# Vin.ID — PWA / Add to Home Screen Notes

Vin.ID supports basic **Add to Home Screen** with a standalone display — enough
for beta testers to launch it like an app. There is **no offline mode** and **no
push notifications** (out of scope).

## What's included

- **Web manifest** at `/manifest.webmanifest`
  ([app/manifest.ts](../app/manifest.ts)): name/short_name **Vin.ID**,
  `display: standalone`, `start_url: /dashboard`, `background_color #111111`,
  `theme_color #E85D24`, `orientation: portrait-primary`, and 192/512 icons
  (incl. a maskable icon).
- **Icons** in [`/public/icons/`](../public/icons/) — simple placeholder marks
  (dark background, orange diamond), generated by
  [scripts/gen-icons.mjs](../scripts/gen-icons.mjs) (dependency-free PNG encoder;
  re-run with `node scripts/gen-icons.mjs`). Replace with a real logo later.
- **Metadata** in [app/layout.tsx](../app/layout.tsx): `manifest`,
  `appleWebApp` (capable + title), apple-touch-icon, and `themeColor` (viewport).
- **In-app install guidance** ([components/pwa/install-instructions.tsx](../components/pwa/install-instructions.tsx)) —
  short, platform-aware copy shown on the dashboard onboarding checklist and in
  **Settings → Install app**. It renders nothing when already running standalone.

## How to test

### iOS Safari (Add to Home Screen)
1. Open the production URL in **Safari** (iOS install only works in Safari).
2. Tap the **Share** button → **Add to Home Screen** → Add.
3. Launch from the new home-screen icon → it opens **standalone** (no Safari UI).

### Android Chrome (Install)
1. Open the production URL in **Chrome**.
2. Tap the **⋮ menu** → **Add to Home screen** / **Install app** (Chrome may also
   show an install prompt automatically).
3. Launch from the icon → standalone window.

### Verify the icon
- The home-screen icon shows the dark/orange mark (not a generic globe).
- Android: DevTools → **Application → Manifest** lists name, theme color, and the
  192/512 icons with no errors; "Installability" has no blocking issues.

### Verify standalone mode
- Launched from the home screen, there's no browser address bar.
- In standalone, the in-app **Install app** guidance disappears (it detects
  `display-mode: standalone`).

## Known limitations

- **No offline mode** — there is no service worker / caching yet.
- **No push notifications.**
- `start_url` is `/dashboard`, which requires auth; if you're signed out when you
  launch, you'll land on the login page (expected).
- Icons are **placeholder** art — swap in a real logo before any public launch.
- iOS "Add to Home Screen" is **Safari-only**; other iOS browsers can't install.
